# Learning Paths

Use this page when you want a role-shaped route through software engineering instead of traversing the whole track in section order.

## Backend Path

### Goal

Build confidence in contracts, testing, concurrency, and safe delivery for API and service work.

### Order

1. `docs/software-engineering/tooling`
2. `docs/software-engineering/apis`
3. `docs/software-engineering/testing`
4. `docs/software-engineering/security-basics`
5. `docs/software-engineering/concurrency`
6. `docs/software-engineering/observability`
7. `docs/software-engineering/reliability`
8. `docs/software-engineering/system-design`
9. `docs/software-engineering/capstones`

### Focus

- request and schema contracts
- regression resistance
- queues, retries, and deadlines
- SLOs, alerts, and rollback

## Platform / Reliability Path

### Goal

Build strength in production behavior, change safety, and team/platform leverage.

### Order

1. `docs/software-engineering/tooling`
2. `docs/software-engineering/observability`
3. `docs/software-engineering/reliability`
4. `docs/software-engineering/performance`
5. `docs/software-engineering/platform-and-delivery`
6. `docs/software-engineering/system-design`
7. `docs/software-engineering/capstones`

### Focus

- signal quality
- overload and blast radius
- rollout and rollback discipline
- platform interfaces and cognitive load

## AI Product Engineer Path

### Goal

Build the judgment needed to ship AI-assisted features safely without confusing fast implementation for finished engineering.

### Order

1. `docs/software-engineering/tooling`
2. `docs/software-engineering/testing`
3. `docs/software-engineering/security-basics`
4. `docs/software-engineering/typescript`
5. `docs/software-engineering/apis`
6. `docs/software-engineering/reliability`
7. `docs/software-engineering/capstones`

### Focus

- spec-first prompting
- generated-code review gates
- runtime validation
- regression and rollout control

## Use Rule

- Pick one path and finish at least one capstone before switching.
- If your work is mostly service code, start with the backend path.
- If your work is mostly operations or release safety, start with the platform / reliability path.
- If your work is mostly feature delivery with AI assistance, start with the AI product engineer path.
